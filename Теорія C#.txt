Теорія C#

	Три слона ООП
	
Інкапсуляція — це об’єднання даних і методів, які з ними працюють, в один об’єкт та обмеження прямого доступу до цих даних ззовні.

Простіше:

Дані «ховаються» всередині об’єкта, а доступ до них відбувається тільки через дозволені методи.

Приклад: замість прямого доступу до змінної — виклик методу.

Наслідування

	Наслідування — це механізм, за допомогою якого один клас (дочірній) отримує властивості та методи іншого класу (батьківського).

Простіше:

Один клас «успадковує» функціональність іншого, щоб не писати той самий код двічі.

Це дає: повторне використання коду та логічну ієрархію.

	Поліморфізм — це можливість використовувати один і той самий інтерфейс для різних типів об’єктів, які реалізують його по-різному.
	

Простіше:

Одна й та сама команда може працювати по-різному для різних об’єктів.

Приклад: метод draw() у різних фігур (коло, квадрат) робить різні речі.


==================================================
1. ОСНОВИ ООП ТА СТРУКТУРА КЛАСІВ
==================================================

[Додано] Access Modifiers (Модифікатори доступу)
Це ключові слова, що налаштовують видимість.
Основні:
- public: доступно всім.
- private: доступно тільки всередині цього класу.
- protected: доступно класу та його спадкоємцям.
- internal: доступно в межах одного проєкту.

Class (Клас)
Це шаблон (креслення), за яким створюються
об'єкти. Клас описує стан (змінні) та
поведінку (методи).
Приклад: Клас "Авто", об'єкт - твоя Peugeot 508.

Property (Властивість)
Це "розумне поле". Механізм доступу до приватних
змінних класу через методи get (читання) та
set (запис). Дозволяє контролювати, які дані
входять або виходять.

[Додано] Constructor (Конструктор)
Спеціальний метод, який викликається автоматично
при створенні об'єкта. Використовується для
початкового налаштування полів.

Struct (Структура)
Схожа на клас, але це Value Type (значимий тип).
Живе в стеку (швидка пам'ять). Використовується
для легких об'єктів (координати, точки), де не
потрібне наслідування.

Partial (Частковий клас)
Ключове слово, що дозволяє розбити код одного
класу на кілька файлів. При компіляції вони
збираються в один. Зручно для великих класів
або автогенерованого коду.

Static (Статичний)
Означає, що елемент належить самому класу, а не
конкретному об'єкту.
Приклад: Math.Pow(). Тобі не треба створювати
об'єкт "Математика", щоб піднести до степеня.

Static Constructor (Статичний конструктор)
Конструктор, який викликається лише один раз —
при першому зверненні до класу. Потрібен для
ініціалізації статичних полів.

==================================================
2. МЕТОДИ ТА ПАРАМЕТРИ
==================================================

Ref
Ключове слово, що дозволяє передати змінну в
метод за посиланням. Метод працює з оригіналом
змінної, а не з копією.
Умова: Змінна має бути ініціалізована ДО
передачі.

Out
Схоже на Ref (повертає дані з методу), але
змінну НЕ обов'язково ініціалізувати до
передачі. Метод ЗОБОВ'ЯЗАНИЙ записати в неї
значення.

Extension Method (Метод розширення)
Дозволяє додавати нові методи до існуючих типів
без зміни їхнього коду. Це статичний метод у
статичному класі з ключовим словом `this`.
Приклад: створити метод .ToMyFormat() для string.

==================================================
3. НАСЛІДУВАННЯ ТА ПОЛІМОРФІЗМ
==================================================

Inheritance (Наслідування)
Механізм, де один клас (Нащадок) переймає поля
і методи іншого класу (Батька). Дозволяє не
писати код двічі.
Синтаксис: class Dog : Animal { }

[Додано] Virtual / Override
- virtual: дозволяє методу бути зміненим у
нащадках.
- override: власне переписує цей метод у
нащадку.

Abstract Class (Абстрактний клас)
Це незавершений клас. Створити його екземпляр
(new AbstractClass()) не можна. Він може містити
як реалізовані методи, так і абстрактні (без
коду), які нащадки мусять реалізувати самі.

Interface (Інтерфейс)
Це контракт. Він містить лише список методів і
властивостей, але БЕЗ реалізації (до C# 8.0).
Клас, що підписує контракт (імплементує),
зобов'язаний реалізувати ВСЕ, що там описано.

Abstract Class vs Interface (Різниця)
1. Наслідування: Клас може мати лише одного
"батька" (абстрактний клас), але багато
інтерфейсів.
2. Стан: Абстрактний клас може мати поля і
конструктори. Інтерфейс — ні (традиційно).
3. Суть: Абстрактний клас — це "хто це є"
(Тварина), Інтерфейс — це "що це вміє"
(ILilyable - Літаючий).

Implementation (Імплементація/Реалізація)
Процес написання коду для методів, які були
оголошені в інтерфейсі або абстрактному класі.

Standard Interfaces (Стандартні інтерфейси)
Вбудовані в .NET контракти.
- IEnumerable: дозволяє перебирати колекцію
(foreach).
- IDisposable: для очищення ресурсів.
- IComparable: для сортування об'єктів.

Aggregation (Агрегація)
Відношення "Частина-Ціле". Частина може існувати
окремо від цілого.
Приклад: "Машина" і "Водій". Якщо машину
знищити, водій залишиться живим.

Composition (Композиція)
Жорстке відношення "Частина-Ціле". Частина НЕ
може існувати без цілого.
Приклад: "Будинок" і "Кімната". Знищиш
будинок — зникнуть і кімнати.

==================================================
4. ПОДІЇ ТА ДЕЛЕГАТИ
==================================================

Delegate (Делегат)
Це змінна, яка зберігає посилання на метод.
Типу як вказівник на функцію. Дозволяє
передавати методи як параметри.

Callback Function (Функція зворотного виклику)
Це функція, яка передається в іншу функцію і
викликається там після завершення якоїсь дії.
Реалізується через делегати.

Lambda Expressions (Лямбда-вирази)
Скорочений запис анонімних методів.
Синтаксис: (вхід) => (вихід).
Приклад: x => x * x (повертає квадрат числа).

Event (Подія)
Механізм повідомлення про те, що щось сталося.
Базується на делегатах. Клас-видавець
"стріляє" подією, а підписники реагують.
Приклад: Button.Click.

==================================================
5. КОЛЕКЦІЇ ТА ДАНІ
==================================================

Collection (Колекція)
Загальна назва контейнерів для зберігання групи
об'єктів. Бувають звичайні (ArrayList) та
узагальнені (List<T>).

Generics (Узагальнення / Дженеріки)
Спосіб писати код, який працює з різними типами
даних, не втрачаючи типізацію.
Приклад: List<T>. Замість T ми підставляємо
int, string або свій клас.
List<int> гарантує, що там лише числа.

Dictionary (Словник)
Колекція пар "Ключ - Значення". Доступ до
елемента здійснюється дуже швидко по ключу.
Приклад: PhoneBook["Ivan"] поверне номер.

Boxing / Unboxing (Упаковка / Розпаковка)
- Boxing: Перетворення значимого типу (int) у
посилальний (object). Дані копіюються зі Стека
в Купу. Повільно.
- Unboxing: Зворотній процес.
Краще уникати, використовуючи Generics.

[Додано] LINQ (Language Integrated Query)
Мова запитів до колекцій. Дозволяє фільтрувати,
сортувати та змінювати дані в стилі SQL.
Приклад: list.Where(x => x > 5).ToList();

[Додано] Enum (Перелічення)
Тип даних, що складається з набору іменованих
констант.
Приклад: Days.Monday, Days.Tuesday.

==================================================
6. ОБРОБКА ПОМИЛОК
==================================================

Exception (Виняток)
Подія, яка виникає при помилці (ділення на 0,
файлу немає). Це об'єкт, що "викидається" і
ламає програму, якщо його не зловити.

[Додано] Try-Catch-Finally
Конструкція для обробки винятків.
- try: код, де може бути помилка.
- catch: що робити, якщо помилка сталася.
- finally: код, що виконається завжди (наприклад,
закриття файлу).

==================================================
7. АТРИБУТИ ТА МЕТАДАНІ
==================================================

Attributes (Атрибути)
Спеціальні теги [...], які додають метадані до
класів, методів чи полів. Самі по собі нічого
не роблять, їх читають інші програми або
бібліотеки.

Data Annotation
Набір стандартних атрибутів для валідації даних.
Часто використовуються в Web (ASP.NET) та EF.
Приклад: [Required], [MaxLength(50)].

Regex (Регулярні вирази)
Потужний інструмент для пошуку та маніпуляцій з
текстом за шаблоном.
Приклад: перевірити, чи схожий текст на email.

==================================================
8. БАЗИ ДАНИХ ТА ORM
==================================================

ORM (Object-Relational Mapping)
Технологія, яка зв'язує базу даних (таблиці) з
ООП кодом (об'єктами). Тобі не треба писати
SQL, ти працюєш з класами.

Entity Framework (EF)
Найпопулярніша ORM для .NET. Дозволяє
працювати з БД через контекст даних.

Database First
Підхід, коли спочатку створюється База Даних
(SQL), а потім EF генерує класи C# на її
основі.

[Виправлено] Code First (Замість Model First)
Сучасний підхід. Ти пишеш класи C# (Entities),
а EF сама створює базу даних і таблиці.
"Model First" — застарілий підхід малювання
схем у дизайнері VS.

Entities (Сутності)
Класи, які відображають таблиці в базі даних.
Один об'єкт Entity = один рядок у таблиці.

One to Many (Один до багатьох)
Тип зв'язку.
Приклад: В одній "Групі" навчається багато
"Студентів". У студента лише одна група.

Many to Many (Багато до багатьох)
Тип зв'язку.
Приклад: Один "Студент" відвідує багато
"Курсів". Один "Курс" відвідують багато
студентів. Потребує проміжної таблиці в БД.

==================================================
9. АСИНХРОННІСТЬ (Важливо знати)
==================================================

[Додано] Async / Await
Ключові слова для асинхронного програмування.
Дозволяють програмі не "зависати", поки
виконується довга операція (запит до БД,
завантаження файлу).
- async: позначає метод як асинхронний.
- await: чекає завершення задачі, не блокуючи
потік.
